Bases de Datos

Introducción

Definición de base de datos: Programa capaz de almacenar gran cantidad de datos, relacionados y estructurados, que pueden ser consultados rápidamente de acuerdo con las características selectivas que se deseen.

Objetivos de las bases de datos: almacenar gran número de información de una forma organizada para su futura consulta, realización de busquedas, nuevo ingreso de datos, etc.

Componentes de un sistema de bases de datos:

  • Tablas: Para guardar datos.
  • Consultas: Para acceder a la información contenida en una base de datos. Con las consultas se pueden modificar, agregar, eliminar y mostrar datos.
  • Informes: Para presentar información contenida en una base de datos. Esta información se muestra de una forma en concreto y en un determinado formato para que pueda ser imprimida.
  • Formularios: Para ingresar datos.
  • Macros: Para crear instrucciones y ejecutar funciones.

    • Modelos de datos

    Modelo conceptual de datos: Es una representación de un sistema, hecho de la composición de conceptos que se utilizan para ayudar a las personas a conocer, comprender o simular un tema que representa el modelo, incluye las entidades importantes y las relaciones entre ellos.

    Modelo lógico de datos: Es un modelo que no es específico de una base de datos que describe aspectos relacionados con las necesidades de una organización para recopilar datos y las relaciones entre estos aspectos.

    Modelo físico de datos: Es un modelo específico de bases de datos que representa objetos de datos relacionales (por ejemplo, tablas, columnas, claves principales y claves externas) y sus relaciones.

    Modelo de datos relacional: Es una colección de información que organiza datos en relaciones predefinidas, en la que los datos se almacenan en una o más tablas (o "relaciones") de columnas y filas.

    Modelo de datos jerárquico: Es un tipo de sistema de gestión de bases de datos que, como su nombre indica, almacena la información en una estructura jerárquica que enlaza los registros en forma de estructura de árbol, en donde un nodo padre de información puede tener varios nodos hijo, y así sucesivamente.

    Modelo de datos de red: Es un modelo de base de datos concebido como un modo flexible de representar objetos y su relación.

    Lenguajes de consulta

    Lenguajes de consulta de datos: Son herramientas que permiten a los usuarios recuperar información de bases de datos y sistemas de gestión de datos de manera estructurada y eficiente.

  • SQL: Lenguaje estándar para bases de datos relacionales.
  • NoSQL Query Languages: Lenguajes específicos para bases de datos NoSQL.
  • SPARQL: Para consultar datos en la web semántica y RDF.
  • XQuery: Recupera y transforma datos en XML.
  • Datalog: Usado en programación lógica y bases deductivas.
  • GraphQL: Acceso flexible a datos en servicios web y API.

    • SQL (Structured Query Language) es un lenguaje de programación utilizado para administrar y consultar bases de datos relacionales. Permite realizar operaciones como la recuperación, inserción, actualización y eliminación de datos en tablas de bases de datos. SQL se utiliza ampliamente en sistemas de gestión de bases de datos relacionales como MySQL, PostgreSQL, Oracle, Microsoft SQL Server y muchos otros.

    Lenguajes de definición de datos (DDL, por sus siglas en inglés, Data Definition Language) son un conjunto de comandos y sentencias utilizados para definir, gestionar y modificar la estructura y la organización de las bases de datos. Estos lenguajes permiten a los administradores de bases de datos y a los desarrolladores especificar cómo se deben crear, modificar y eliminar tablas, índices, restricciones y otros elementos de una base de datos

    Diseño de bases de datos

    Proceso de diseño de bases de datos: Es el conjunto de pasos y decisiones tomadas para crear una estructura organizada y eficiente que almacene y gestione datos de manera coherente. Incluye la definición de tablas, relaciones, atributos y restricciones para satisfacer las necesidades de almacenamiento y recuperación de información de una aplicación o sistema.

    Diseño conceptual de bases de datos: Es la etapa inicial del proceso de diseño de bases de datos. En esta fase, se crea un modelo abstracto de la base de datos que se centra en las entidades, sus atributos y las relaciones entre ellas, sin preocuparse por detalles técnicos o estructuras físicas de almacenamiento. El objetivo es comprender y representar de manera clara y completa la estructura lógica de la información que se desea almacenar y gestionar en la base de datos.

    Diseño lógico de bases de datos: Se refiere a la fase del proceso de diseño de bases de datos en la que se traduce el modelo conceptual en una estructura lógica que puede implementarse en un sistema de gestión de bases de datos (DBMS). En esta etapa, se definen las tablas, atributos, claves primarias, claves foráneas y otras restricciones necesarias para representar la estructura de datos de manera coherente y eficiente, teniendo en cuenta las capacidades y requisitos del DBMS específico que se utilizará. El diseño lógico se enfoca en los detalles técnicos y la implementación práctica de la base de datos.

    Diseño físico de bases de datos: Es la fase final del proceso de diseño de bases de datos. En esta etapa, se traducen las estructuras lógicas y las especificaciones del diseño lógico en una implementación física concreta en el sistema de gestión de bases de datos (DBMS) elegido. Esto implica decisiones sobre la organización de datos en disco, índices, tamaños de campo, tipos de datos y otros aspectos técnicos que afectan directamente al rendimiento y la eficiencia de la base de datos. El diseño físico se enfoca en la optimización y la preparación para la implementación y la administración de la base de datos en un entorno real.

    Administración de bases de datos

    Administración de usuarios y permisos: Se refiere al proceso de gestionar quiénes tienen acceso a un sistema o recurso informático y qué acciones específicas están autorizados a realizar. En un contexto de bases de datos, esto implica controlar quién puede acceder a la base de datos, qué datos pueden ver o modificar, y qué operaciones pueden realizar. Esto se logra mediante la creación y gestión de cuentas de usuario, roles y permisos que definen las acciones y los recursos a los que se tiene acceso. La administración de usuarios y permisos es fundamental para garantizar la seguridad y la integridad de los datos almacenados en una base de datos.

    Administración de seguridad: Se refiere a la gestión de medidas y políticas diseñadas para proteger un sistema, una red, una base de datos u otros recursos informáticos contra amenazas y riesgos potenciales. Esto incluye la implementación de controles de seguridad, la asignación de permisos y roles adecuados, la supervisión de actividades sospechosas y la aplicación de políticas de seguridad. Puede abarcar aspectos como la gestión de contraseñas, el cifrado de datos, la autenticación de usuarios, la detección y respuesta a incidentes, y la aplicación de parches de seguridad, entre otros.

    Administración del rendimiento: Se refiere a las prácticas y estrategias utilizadas para garantizar que una base de datos funcione de manera eficiente y cumpla con los requisitos de rendimiento esperados. Esto incluye la optimización de consultas, el diseño adecuado de índices, la asignación de recursos del sistema y la monitorización continua para identificar y resolver problemas que puedan afectar la velocidad y la capacidad de respuesta de la base de datos.

    Administración de la integridad de los datos: Se refiere a asegurar que los datos sean precisos, coherentes y fiables. Esto implica establecer reglas, validar entradas, controlar el acceso y realizar respaldos para evitar errores o corrupción de datos.

    Administración de la replicación de datos: Se refiere a la gestión de copias sincronizadas de datos en múltiples ubicaciones para garantizar la disponibilidad y la redundancia. Esto permite que los datos estén disponibles en caso de fallos o necesidades de acceso rápido en diferentes lugares.

    Administración de la recuperación de desastres: Se refiere a la planificación y las estrategias para restaurar los datos y la funcionalidad de la base de datos en caso de eventos catastróficos, como fallos de hardware, errores humanos o desastres naturales. El objetivo es asegurar la continuidad del negocio y la protección de los datos críticos.

    Técnicas de implementación de sistemas de bases de datos

    Hardware de bases de datos: Se refiere a los componentes físicos, como servidores, discos duros, memoria RAM y unidades de procesamiento, que forman la infraestructura tecnológica utilizada para almacenar, gestionar y acceder a los datos en una base de datos. El rendimiento y la capacidad de una base de datos están influenciados significativamente por la calidad y la configuración del hardware que la respalda.

    Software de bases de datos: Es un programa informático o un conjunto de programas diseñados para crear, gestionar y manipular bases de datos. Estos software permiten la creación de tablas, relaciones, consultas, informes y la gestión de datos de manera eficiente. Ejemplos comunes de software de bases de datos incluyen Microsoft SQL Server, Oracle Database, MySQL, PostgreSQL y MongoDB.

    Sistemas de bases de datos relacionales: Son software diseñados para gestionar bases de datos que siguen el modelo de datos relacionales. En este tipo de sistemas, los datos se organizan en tablas (o relaciones) que están interconectadas mediante claves primarias y claves foráneas. Algunos ejemplos de DBMS relacionales populares incluyen Microsoft SQL Server, Oracle Database, MySQL, PostgreSQL, y SQLite. Estos sistemas permiten el almacenamiento, consulta y manipulación de datos de manera estructurada y eficiente, y son ampliamente utilizados en aplicaciones empresariales y en la mayoría de los sistemas de gestión de datos.

    Sistemas de bases de datos NoSQL: Son un tipo de software de gestión de datos diseñados para manejar datos no estructurados o altamente distribuidos, ofreciendo flexibilidad y escalabilidad en comparación con las bases de datos relacionales tradicionales.
